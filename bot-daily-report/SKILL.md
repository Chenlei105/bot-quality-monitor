# Bot Daily Report

**Bot 健康度日报生成器 v5.0**

---

## 概述

Bot Daily Report 是 Bot Quality Monitor 系统的报告生成层，基于四维指标体系生成健康度日报：

- 📊 **四维健康分** - 质量(50%) + 效率(30%) + 资源(20%)
- 🚨 **异常红灯预警** - 独立于健康分，触发即告警
- 📬 **移动端风格日报** - 飞书卡片推送，按图片格式设计
- 🔔 **智能信号** - 高分低用/低分高风险/高风险场景

---

## 核心功能

### 1. 四维健康分计算

根据指标体系文档，健康分由三个维度加权计算：

```
健康分 = 质量分 × 50% + 效率分 × 30% + 资源分 × 20%
```

#### 质量维度 (50%)
- **解决率** (30%): 任务已解决比例
  - 判定：用户无追问、自然结束、采纳回复 = 已解决
  - 判定：用户重说、点击重新生成、重复追问 = 未解决
- **满意度** (20%): 从行为信号推算
  - 正向信号：采纳/复制/分享、点赞、深入探索
  - 负向信号：追问/重试次数、主动取消、点踩、辱骂

#### 效率维度 (30%)
- **任务解决时长**: 最后一轮回复时间 - 第一轮请求时间
- **有效时长**: 扣除用户主动思考间隔
- **评分**: 与同复杂度任务基准比较

#### 资源维度 (20%)
- **单次 Token 消耗**: 完成一个任务消耗的总 Token
- **评分**: 按任务类型设定基准线

#### 异常维度（不参与健康分，独立红灯）
- **非预期中断率**: <1% 正常，1-3% 注意，>3% 警报
- **P99 响应时长**: <基准2x 正常，2x-5x 注意，>5x 警报

### 2. 评级标准

```
健康分 ≥ 90 分: 🟢 优秀
健康分 ≥ 75 分: 🟡 良好
健康分 ≥ 60 分: 🟠 待改善
健康分 < 60 分: 🔴 需关注
```

## 三类智能信号

| 信号类型 | 检测条件 | 建议 |
|----------|----------|------|
| 🟢 高分低用 | 健康度≥85 且 周使用≤5次 | Bot 质量很好但用得太少 |
| 🔴 低分高风险 | 纠错率≥10% 或 失败≥5次 | 错误率太高，立即优化 |
| ⚠️ 高风险场景 | 失败率≥30% 且 样本≥5 | 该场景失败率过高 |

```json

    "efficiency": 0.30,
    "resource": 0.20
```

```json
{
  "signalThresholds": {
```
